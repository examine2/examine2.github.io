  根据提示，我们要通过给出的输入框访问http://web1/flag 拿到flag，但输入框对输入的URL做了限制
  下载源文件，主要看对URL的处理部分的代码：

  函数check_hostname()主要应用了Python的切片，概况起来有以下几点：

  • URL以”http://“开头
  • 截取保留URL”@”字符之后、”:”和”/“字符之前的部分
  • 截取下来的字符串与whitelist_hostname中预留的字符串对比，如果在里面，则执行URL的访问功能，否则返回”Not in whitelist”

  既然要访问http://web1.flag，那么我们构建的URL肯定就是以http://web1.flag开头的。但构建的URL经处理后必须只剩下”example.com”或”www.example.com"
  我主要是利用了源Python代码对URL的处理中，会删除”@”之前的、只保留”@”后面的字符串；于是尝试把URL构造成：

http://web1.flag@example.com

  但是这本身就不是一个有效的URL，根本无法解析
  向URL中写入一些乱七八糟的东西，但是却保持URL的正常解析。一个方法就是将其作为POST的参数值，传过去
  随便构造一个参量名为a，值为@example.com；于是URL就被构造成了：

http://web1/flag?a=@example.com

  解析成功、访问成功，拿到flag

  但是取消代理准备去锤龙的时候，发现Attack被修改回原来的值了

不同寻常的Python考试

  本来以我的实力，做完上面的题已经无题可做了，但是我阅读了Python的源文档后，发现我还是有可能做出来这题的…于是一度沉迷，晚上12点看着这道题看到2点
  代码的逻辑很简单，共20道Python小题目，输入题号，再输入一个字符串，字符串由ast.literal_eval()处理，经处理后的字符串如果能够让该题号对应的Python方法返回True，则成功解决一题
  整道题在解决10、15、20道小题的地方分别设置了3个flag
  上网搜ast.literal_eval(),发现几乎所有的教程都是将它与eval()一起讲的
  eval()会将字符串类型的list、tuple、dict转变成原有的类型，如eval('[1,2,3]') = [1,2,3]，数据类型从string变成了list；此外，eval()还能够对解析的字符串做处理，如eval('print("Hi")')等效于直接执行语句print("Hi")
  这也意味着eval()隐藏着巨大的风险，倘若eval()被恶意传入删除文件等的字符串代码，后果不堪设想
  通俗的讲，ast.literal_eval()就是安全的eval()，它会对传入的参数进行检查，判断执行后得到的结果是否属于Python的合法类型，不符合则抛出异常
  在本题中，ast.literal_eval()的作用就是解析字符串成Python数据类型，再进行后面运算
  下面来我做出来的10小题

  这一题属于签到题，用于熟悉规则

  这题考察Python的==和is的差异，==只检查数值上是否相同，而is除了数值外，还好额外检查类型，两者都相同才返回True

  这题在亲自运行了一下reversed()函数后才发现它返回的是一个生成器(generator)

  生成器在Python中比较简洁的创建方法是L = (x for x in range(1,11))，这样就创建了一个含1-10的生成器
  比较容易混淆的是L = [x for x in range(1,11)]，这句代码返回的是一个list，一个存储有1-10的list
  两者最大的不同就是生成器中的元素是当需要用到的时候才生成的，而list中的元素是早已生成、在内存中分配对应空间的
  如果要求计算1到1000亿之间所有数的总和，用L = [x for x in range(1,11)]语句是会失败的，因为光创建一个长度为1000亿的list，Python就不会允许，将强制中断
  反观生成器，一个含1到1000亿的生成器是有可能计算出总和的，因为每次都是临时生成下一个需要的数据，占用内存空间不大。虽然执行得非常慢，但终究能成功
  生成器最大的特点就是一次性，一个含1到100的生成器，某次执行代码遍历了生成器的1-50元素，那么下次生成器生成的将会是51，前面已生成的元素直接就没了

  回到题目，既然r是生成器，第一次执行list(r)得到r的list形式，第二次执行list(r)后得到一个空列表[]，也就是[3,2,1] = [] + answer，answer就是[3,2,1]

  这题乍一眼看上去就是乘法分配律，但考察的是字符串与整数也可以进行运算

  很显眼的乘法结合律，同样将字符串传入，搅浑水，字符串如果与负数相乘，将得到空字符串""

  这题可以说是最没有难度的一题了，幂指数能够得到整数也能够得到小数
  接下来的题目就真的超出范围了，无力…

  这题的答案是经过不断的尝试，试出来的
  当answer为字符串时，如果先执行answer in answer，返回的布尔值再进行boolean == answer，但是无论如何都会判定为False；如果先执行answer == answer，那么接下来就是answer in boolean，直接报错
  将两个运算符分开来怎么赋值都是错的，但将它们结合到一起，只需一个字符串，就能返回True

  这题明显违反常识，max()只返回传入参数中最大的，怎么可能调换一下传入参数的顺序，产生的结果就不同？！
