学习使用命令行工具curl

curl –(Client URL)

一个利用URL语法在命令行下工作的文件传输工具

介绍

基础用法：
$ curl https://www.baidu.com
上面的命令便是curl最基本的用法，获取URL处的源代码

curl往往携带许多参数，参数也是curl的精髓，没有参数的curl只能获取源代码，useless

以参数-A为例，-A能够指定客户端的用户代理标头，即User-Agent，伪造User-Agent用于伪造浏览器；在一些CTF题目中，往往需要使用指定的浏览器去访问网页才能得到flag，而真的去下载该浏览器是傻子行为，有时候那些浏览器根本不存在

参考pico2019Web的一道题目——‘picobrowser’，要求我们使用一款名为picobrowser的浏览器去访问网站https://2019shell1.picoctf.com/problem/37829/flag，即可得到flag

那么使用curl：
$ curl -A picobrowser https://2019shell1.picoctf.com/problem/37829/flag
服务器将把你看作你是用picobrowser访问网页的，就会返回对应的网页

在网页源代码中仔细查找，即可找到picoCTF{xxx}

这里还有一点值得注意，curl在命令行下即可运行，也就是Windows和Linux都行

将上面含-A的curl命令在Windows的cmd窗口中输入，同样能够返回想要的源代码，同样能够找到picoCTF{xxx}。但是使用Linux的命令配合curl，将会使curl得到更好的效果

比如，如果返回的源代码太多，我们不会想去自己一行一行地搜索有没有picoCTF{xxx}，这时可以利用Linux的grep命令，用|形成管道，将源代码传递给grep作为正则匹配文本，直接匹配：
$ curl -A picobrowser https://2019shell1.picoctf.com/problem/37829/flag | grep pico

便捷高效

因此练习curl在Linux系统下最好（至少我是）

以修改User Agent的-A为起点，正式展开对curl的学习

部分概念

参数位置

在curl中，命令行解析器总是解析整个命令行，因此参数选项可以放在整个命令行的任意位置：
curl https://2019shell1.picoctf.com/problem/37829/flag -A picobrowser | grep pico

当然你要是说要把参数放在curl之前，可别怪我一砖拍下去

所谓的任意位置也只限于curl命令，grep命令接收curl传送过来的文本，因此必须放在|之后

短形式 & 长形式

curl的参数大多是为了追求高效率的缩写，如-A，称为短形式；但当参数过多，单独一个字母的参数反倒容易引起困惑（比如，-a，它是一个与-A没关系的参数），短形式也不能从字面意思上记忆

所有短形式都对应着一个长形式，例如-A，它的长形式就是--user-agent

短形式用一个中划线-指定，而长形式用两个中划线指定--。两者是完全互相等效的

于是上面的picobrowser题也可以写成：
curl --user-agent picobrowser https://2019shell1.picoctf.com/problem/37829/flag | grep pico

当你实在忘记修改UA(User Agent)的参数缩写是什么的时候，只要你还记得User Agent这个词语，就能敲上--user-agent

单引号 or 双引号

多数情况下，在curl中添加单引号或双引号是无关痛痒的。下面三条命令产生的效果完全一样：
curl -A picobrowser https://2019shell1.picoctf.com/problem/37829/flag | grep pico
curl -A 'picobrowser' https://2019shell1.picoctf.com/problem/37829/flag | grep pico
curl -A "picobrowser" https://2019shell1.picoctf.com/problem/37829/flag | grep pico

单引号和双引号都将内容概括成一个字符串，两者没有任何区别

当参数包含空格的时候，引号的作用就凸显出来了：假如要求你使用一个名为’Are you OK?’的浏览器访问页面，你必须写成：
curl -A "Are you OK?" https://2019shell1.picoctf.com/problem/37829/flag | grep pico

不添加引号，Are将会被单独解析为UA(User Agent，使用缩写是个好习惯)，而剩下的you和OK?很可能导致未知的错误

Help

当你需要查阅一些关于curl参数的文档的时候，除了一些概括性的博客文章，还可以查阅内置的官方文档

$ curl --help将显示所有参数的短形式、长形式以及简要用法

$ curl --manual将详细地呈现每个参数的使用方法

部分参数汇总

 -A --user-agent

设置用户代理(User Agent)，简称UA；通常用于伪装成特定的浏览器

一些网站常常通过判断UA来给不同的操作系统、不同的浏览器发送不同的页面，因此可能造成某些页面无法在某个浏览器中正常显示。可以通过伪装UA来绕过检测

在网上可以找到大部分浏览器的UA字符串

 -e --referer

与设置UA的-A类似，-e设置的是来源网址(Referer)

当你从www.google.com点击进入www.examine2.top，和从www.baidu.com点击进入www.examine2.top时，这两次的请求头信息是不完全一样的，至少两次的Referer分别是www.google.com和www.baidu.com

Referer通常用于统计网站流量来源，有时也用于防盗链。指定我的图片服务器只能我自己的网站访问，那么图片服务器每次都会判断取到的Referer是否合法

空Referer的产生是由直接从浏览器的地址栏输入URL访问导致的

Referer的正确英语拼法是referrer，由于早期HTTP规范的拼写错误，为了保持向后兼容就将错就错了。其它网络技术的规范试图修正此问题，使用正确拼法，所以目前拼法不统一

本部分参考

 -H --header

事实上，上面的User-Agent和Referer都属于HTTP请求头，而常用的HTTP请求头就有30多个：

  • Accept
  • Accept-Language
  • Authorization
  • Content-Type
  • Host
  • If-Modified-Since
  • …

在curl中，为每一个请求头都准备一个缩写参数，没必要且容易引起混淆。为此可以通过-H来指定请求头和请求头的内容

$ curl -A picobrowser www.google.com和$ curl -H 'user-agent: picobrowser' www.google.com是等效的

-H参数后接字符串，请求头和请求头内容之间用冒号:和一个空格连接。一个-H参数修改一个请求头信息，要修改多个请求头，建议使用多个-H

攻防世界Web题的新手练习区中，题目”xff_referer”适合练手

题目要求我们访问一个网页，请求头”X-Forwarded-For”(即xff)代表Web服务器最原始的IP地址，伪造它即可实现给出的提示“ip地址必须为123.123.123.123”

后面又要求“必须来自https://www.google.com”，于是我们使用curl，一波带走：

$ curl -H 'x-forwarded-for: 123.123.123.123' -H 'referer: https://www.google.com' http://111.198.29.45:55724/

拿到flag总有一种喜悦是不是，n_n
